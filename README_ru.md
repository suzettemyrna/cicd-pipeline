# CI/CD Pipeline

> English version: [README.md](README.md)

Практический лабораторный проект по изучению непрерывной интеграции и непрерывной доставки приложений.

В рамках проекта настраивается собственный GitLab Runner, создаётся CI/CD-конвейер для проекта на языке C, выполняются автоматическая сборка, проверка кодстайла, запуск интеграционных тестов, деплой на удалённую виртуальную машину и отправка уведомлений о результатах выполнения пайплайна.

Все этапы выполняются средствами GitLab CI/CD и Linux без использования графических инструментов.

## Темы проекта

* GitLab Runner
* GitLab CI/CD
* `.gitlab-ci.yml`
* автоматическая сборка проектов
* артефакты сборки
* `clang-format`
* интеграционные тесты
* SSH и SCP
* автоматический деплой
* Telegram-уведомления

## Результаты выполнения

После выполнения проекта пользователь сможет:

* настраивать и регистрировать GitLab Runner;
* создавать многоэтапные CI/CD-конвейеры;
* автоматизировать сборку приложений;
* сохранять и использовать артефакты сборки;
* выполнять автоматическую проверку кодстайла;
* запускать интеграционные тесты в пайплайне;
* автоматизировать деплой на удалённый сервер;
* настраивать уведомления о результатах выполнения пайплайнов.

## Быстрый старт

Для выполнения лабораторных работ потребуется:

* VirtualBox или аналогичная система виртуализации;
* две виртуальные машины с Ubuntu Server 22.04 LTS;
* GitLab Runner;
* доступ к GitLab-проекту;
* права root/sudo;
* SSH-доступ между виртуальными машинами.

Лабораторные работы рекомендуется выполнять последовательно, начиная с [Part 1](docs/ru/Part1_ru.md).
