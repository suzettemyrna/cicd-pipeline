## Part 5. Этап деплоя

### Поднимем вторую виртуальную машину *Ubuntu Server 22.04 LTS*

![](images/part5/1.png)

На новой машине, как и на старой, обновим все пакеты и устроим общую папку (удобная штука).

После этого настроим статический маршрут между машинами:

1. Выключим обе машины и в настройках обоех добавим сетевой адаптер типа *Внутренняя сеть*:
   
  ![](images/part5/2.png)

2. Настроим файлы *etc/netplan/00-installer-config.yaml* на обеих машинах так, чтобы они соединялись через внутреннюю сеть:

  - Первая машина (сервер, на котором работает **GitLab Runner**):

    ![](images/part5/3.png)

  - Вторая машина (новая):

    ![](images/part5/4.png)

  Не забудем обновить настройки сети с помощью `sudo netplan apply`.

3. Проверим, получилось ли связать компьютеры сетью:

  ![](images/part5/5.png)
  ![](images/part5/6.png)

  Всё пингуется, всё хорошо.

Теперь нужно настроить доступ по **SSH** без пароля с первой виртуальной машины на вторую:

1. Для начала установим *ssh-сервер* (`sudo apt install openssh-server`) и запустим его (`sudo systemctl start ssh`), после чего проверим, работает ли он (`sudo systemctl status ssh`):

  ![](images/part5/7.png)

  Чтобы *ssh-сервер* автоматически запускался при загрузке системы, нужно ввести `sudo systemctl enable ssh`.

2. На второй машине откроем файл */etc/ssh/sshd_config* и раскомментим следующие строки:
   - `Port 22`
   - `PermitRootLogin yes`
   - `PasswordAuthentication yes`

  ![](images/part5/8.png)
  ![](images/part5/9.png)

  Затем перезапускаем **ssh**: `sudo systemctl restart ssh`.

3. Дальше нам нужно перейти на первой машине на пользователя *gitlab-runner*. Дадим ему право делать что угодно и без пароля. Для этого наберём команду `sudo visudo` и добавим там строку `gitlab-runner ALL=(ALL) NOPASSWD: ALL`.

  Переключаемся на пользователя *gitlab-runner* командой `sudo su gitlab-runner`.

4. Создаём на первой машине пару *ssh-ключей* командой `ssh-keygen`.

5. С первой машины передаём публичный ssh-ключ на вторую. По идее это можно сделать с помощью команды `ssh-copy-id username@2nd_machine_ip`, но у меня почему-то не получилось, так что я переносила вручную через общую папку.

  Ключ должен лежать на второй машине в файле *~/.ssh/authorized_keys*:
   
  ![](images/part5/10.png)

6. Теперь можно попробовать подключиться к второй машине по ssh без пароля: `ssh user@192.168.1.2`:

  ![](images/part5/11.png)
