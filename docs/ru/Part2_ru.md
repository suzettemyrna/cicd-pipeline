## Part 2. Сборка

Напишем этап для **CI** по сборке приложений из проекта *C3_SimpleBashUtils*.

- Не забудем сгенерировать ssh-ключ на виртуальной машине и добавить его в список ключей на *GitLab*. После чего устанавливаем права на ключи (иначе не клонируется): `chmod 700 ~/.ssh`, `chmod 600 ~/.ssh/id_rsa`, `chmod 644 ~/.ssh/id_rsa.pub`. 
  
- Сообщим гиту, кто мы есть: `git config --global user.name <nickname>`, `git config --global user.email <email_address>`.

- После этого клонируем оба проекта на машину. Для этого я написала скрипт для клонирования обоих проектов на виртуальную машину и перехода в них на ветку *develop*. Положила в */src/scripts/*.

- Копируем папки *cat* и *grep* из папки *C3_SimpleBashUtils-5* в папку текущего проекта *DO6_CICD-1/src/*.

- Cоздаём файл _gitlab-ci.yml_ в папке *DO6_CICD-1/src/*.

Подготовка закончена, можно и делом заняться.

### В файле _gitlab-ci.yml_ добавим этап запуска сборки через мейк файлы из проекта _C3_; а файлы, полученные после сборки (артефакты), сохраним в произвольную директорию со сроком хранения 30 дней

Файл _gitlab-ci.yml_ выглядит так:

![](images/part2/1.png)

Объяснение:

  - **stages**: Определяет список этапов (*stages*) для выполнения пайплайна. У нас пока один этап — *build*.
  - **build_job**: Название задачи (*job*), которая будет выполняться на этапе *build*.
    - **stage: build**: Указывает, что задача *build_job* будет выполняться на этапе *build*.
    - **script**: Указывает команды, которые будут выполняться в задаче.
    - **artifacts**: Определяет артефакты (результаты сборки), которые нужно сохранить после выполнения задачи.
      - **paths**: Указывает пути к файлам, которые будут сохранены как артефакты.
      - **expire_in**: Указывает срок хранения артефактов — в нашем случае, 30 дней.
